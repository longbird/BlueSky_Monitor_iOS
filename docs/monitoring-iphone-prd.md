# PRD — SysManager 모니터링 iOS 앱

버전: v0.1 (초안)  
작성일: 2026-02-26  
작성자: OpenClaw

---

## 1. 배경 / 목적
- 기존 Windows용 SysManager의 “모니터링” 기능을 iOS 앱으로 분리 제공.
- 현장/외근 중에도 상태 확인 및 장애 대응을 즉시 수행할 수 있도록 모바일화.
- REST API는 별도 팀에서 구축 중이므로, 본 문서는 앱/요구사항 중심으로 정의.

---

## 2. 목표(성공 기준)
- iOS 앱에서 **센터/서버 상태 요약**을 30~60초 주기로 확인 가능.
- **장애(offline/bad)** 상태가 즉시 시각적으로 식별 가능.
- 상세 화면에서 **CPU/MEM/DISK/Network** 등 핵심 지표 확인 가능.
- 앱 출시 후 **1개월 내 운영팀 업무 대응 시간 20% 이상 단축**.

---

## 3. 범위 (In / Out)
### In Scope
- 모니터링 요약 리스트
- 상세 화면(서버별 상태/지표)
- 상태 필터(정상/경고/장애/오프라인/미사용)
- 자동 갱신 / 수동 새로고침
- 기본 알림(Push) 설계

### Out of Scope
- SysManager의 기타 기능(정산, 고객지원, 공지사항 등)
- 서버 설정 변경/관리 기능(읽기 전용 유지)
- 관리자 계정 생성/권한 관리 UI

---

## 4. 사용자 / 이해관계자
- **운영팀(주 사용자)**: 장애 및 상태 확인
- **관리자/팀장**: KPI 및 안정성 확인
- **개발팀**: API 제공/유지보수

---

## 5. 사용자 시나리오
1) 출근 전, 앱에서 센터별 상태를 확인한다.
2) 빨간 상태(offline/bad)를 클릭해 상세로 진입한다.
3) 서버별 CPU/메모리/네트워크 지표를 확인한다.
4) 필요 시 담당자에게 즉시 연락/조치.

---

## 6. 요구사항 (Functional)
### 6.1 모니터링 요약 화면
- 센터 리스트 표시
- 상태 뱃지 색상
  - good: 녹색
  - normal: 파랑
  - bad: 주황
  - offline: 빨강
  - unused: 회색
- 주요 지표 요약( CPU/MEM/Storage/Network Rx/Tx )
- 정렬: 최신 상태 업데이트 기준

### 6.2 상세 화면
- 서버별 카드/테이블
- 지표: CPU, MEM, DISK, RX/TX, lastUpdated
- 상태 표시
- 차트(옵션, v1.1)

### 6.3 필터
- 상태 필터: 정상/경고/장애/오프라인/미사용
- 센터 필터(옵션)

### 6.4 새로고침
- 자동 30~60초
- 수동 Pull to Refresh

### 6.5 알림(Push)
- Offline 발생 시 Push
- 우선순위: Offline > Bad > Normal

---

## 7. 비기능 요구사항 (Non-Functional)
- iOS 15+ 지원
- 네트워크 불안정 상황에서 재시도
- 보안: 토큰 기반 인증, HTTPS 필수
- 응답 지연 시 로딩 표시
- 장애 발생 시 직관적 UI

---

## 8. 데이터 요건
- REST API 제공 항목 기준
- DB SP: `dbo.sys_monitor_s_1` 기반
- 서버 상태는 socket 프로토콜 → API에서 JSON 제공 예정

- 서버별 섹션
- 상태 뱃지 + 지표 표
